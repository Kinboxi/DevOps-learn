# Глава 3. Перенаправление

> [!info] Зачем это знать
> Перенаправление позволяет управлять потоками данных — куда пишется вывод команды, откуда берётся ввод, как команды передают данные друг другу.

---

## Три потока

Каждый процесс в Linux имеет три стандартных потока:

| Поток | Номер | По умолчанию |
|-------|-------|--------------|
| stdin | 0 | клавиатура |
| stdout | 1 | терминал |
| stderr | 2 | терминал |

> [!tip] Аналогия
> stdin — рот (ввод), stdout — уши (вывод), stderr — крик о помощи (ошибки)

---

## Перенаправление вывода

### command > file.txt
```
> записать stdout в файл (перезапишет файл если существует)
```

### command >> file.txt
```
> добавить stdout в конец файла (не перезапишет)
```

### command 2> error.txt
```
> записать stderr в файл
```

### command 2>> error.txt
```
> добавить stderr в конец файла
```

### command > file.txt 2>&1
```
> записать и stdout и stderr в один файл
```

### command &> file.txt
```
> короткая запись — записать и stdout и stderr в файл
```

### command > /dev/null
```
> выбросить вывод в никуда — подавить stdout
```

### command 2>/dev/null
```
> подавить только ошибки
```

### command &>/dev/null
```
> подавить весь вывод
```

---

## Перенаправление ввода

### command < file.txt
```
> использовать файл как stdin вместо клавиатуры
```

### command << EOF
```
> heredoc — передать многострочный текст как stdin
```

Пример heredoc:
```bash
cat << EOF
строка 1
строка 2
EOF
```

---

## Пайп

Пайп `|` передаёт stdout одной команды в stdin другой.

### command1 | command2
```
> передать вывод command1 на вход command2
```

### ls -la | grep ".txt"
```
> найти .txt файлы в выводе ls
```

### ps aux | grep nginx
```
> найти процесс nginx
```

### cat file.txt | wc -l
```
> посчитать строки в файле
```

### command | tee file.txt
```
> показать вывод в терминале И записать в файл одновременно
```

### command1 | command2 | command3
```
> цепочка команд — можно соединять сколько угодно
```

---

## Подстановка команд

Результат команды можно вставить в другую команду.

### $(command)
```
> выполнить команду и вставить её вывод
```

Примеры:
```bash
echo "Сегодня: $(date)"
mkdir "backup_$(date +%Y-%m-%d)"
file $(which nginx)
```

---

## Полезные команды для работы с потоками

### tee file.txt
```
> читает stdin, пишет одновременно в stdout и в файл
```

### xargs
```
> берёт stdin и передаёт как аргументы следующей команде
```

### find /tmp -name "*.log" | xargs rm
```
> найти все .log файлы и удалить их
```

### cat file.txt | xargs echo
```
> передать содержимое файла как аргументы echo
```

## Практические примеры

```bash
# Сохранить список процессов в файл
ps aux > processes.txt

# Сохранить ошибки nginx в отдельный файл
nginx 2> nginx_errors.txt

# Найти слово в файлах и сохранить результат
grep -r "error" /var/log/ > found_errors.txt

# Посмотреть 10 самых больших файлов
du -sh /* 2>/dev/null | sort -rh | head -10

# Записать вывод и ошибки в разные файлы
command > output.txt 2> errors.txt
```

> [!warning] Важно
> `>` перезаписывает файл без предупреждения. Используй `>>` если хочешь добавить а не перезаписать.
