## ping

Проверяет доступность хоста — отправляет пакет и ждёт ответа. Если ответа нет — хост недоступен или заблокирован.

### ping google.com
```
> проверить доступность хоста
```

### ping -c 4 google.com
```
> отправить ровно 4 пакета и остановиться
```

---

## traceroute / mtr

Показывает через какие роутеры проходит пакет до хоста. Помогает найти где именно обрывается соединение.

### traceroute google.com
```
> показывает весь путь пакета до хоста
```

### mtr google.com
```
> то же самое но в реальном времени — удобнее для диагностики
```

---

## ss / netstat

Показывает открытые порты и активные соединения на сервере. Используешь когда хочешь понять что слушает сервер и кто к нему подключён.

### ss -tuln
```
> все открытые порты на сервере
```

### netstat -tuln
```
> альтернатива ss, старая утилита — делает то же самое
```

---

## telnet / curl

Проверяют подключение к конкретному порту и делают HTTP запросы.

### curl -I url
```
> проверить отвечает ли сервер и какой код возвращает (200, 404, 500)
```

### telnet ip port
```
> проверить открыт ли конкретный порт на хосте
```

---

## nc (netcat)

Универсальный инструмент для работы с сетью. Проверяет порты, передаёт данные, может слушать входящие подключения.

### nc -zv ip port
```
> проверить открыт ли порт — удобнее telnet
```

### nc -zv ip 20-80
```
> проверить диапазон портов
```

---

## nmap

Сканирует открытые порты на хосте. Используешь когда нужно понять что запущено на сервере.

### nmap ip
```
> сканировать открытые порты хоста
```

### nmap -sV ip
```
> сканировать порты и определить версии сервисов
```

---

## tcpdump

Захватывает сетевые пакеты в реальном времени. Используешь для глубокой диагностики — видишь весь трафик который проходит через сервер.

### tcpdump -i eth0
```
> захватить весь трафик на интерфейсе eth0
```

### tcpdump -i eth0 port 80
```
> захватить только трафик на порту 80
```

---

## iftop

Показывает сетевой трафик в реальном времени — кто с кем общается и сколько данных передаёт.

### iftop
```
> запустить мониторинг трафика (нужен root)
```
