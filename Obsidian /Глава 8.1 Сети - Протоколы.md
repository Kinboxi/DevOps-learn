## TCP vs UDP

| | TCP | UDP |
|--|-----|-----|
| Надёжность | да | нет |
| Скорость | медленнее | быстрее |
| Использование | SSH, HTTP, файлы | видеозвонки, игры |

> [!tip] Аналогия
> TCP — заказное письмо с уведомлением. UDP — бросил в ящик и забыл.

---

## HTTP / HTTPS

Протокол браузера. Запрос → ответ.

- **GET** — получить данные
- **POST** — отправить данные

| Код | Значение        |
|-----|-----------------|
| 200 | OK              |
| 301 | перенаправление |
| 404 | не найдено      |
| 500 | ошибка сервера  |

HTTPS — HTTP + шифрование TLS. Порт 443.

---

## TLS / SSL

Шифрует соединение. Уровень 6 OSI. Делает HTTP → HTTPS.

TLS Handshake:
1. Клиент подключается
2. Сервер отдаёт сертификат
3. Договариваются о шифровании
4. Всё дальше зашифровано

> [!info]
> SSL устарел — везде TLS, но говорят SSL по привычке.

## SSH

Удалённое подключение к серверу. Порт 22, TCP.

### ssh user@ip
```
> подключиться к серверу
```

### ssh -p 2222 user@ip
```
> подключиться на нестандартный порт
```

### ssh-keygen
```
> создать пару ключей для входа без пароля
```

### ssh-copy-id user@ip
```
> скопировать ключ на сервер
```

### scp file.txt user@ip:/path/
```
> скопировать файл на сервер по SSH
```

Конфиг:
```
/etc/ssh/sshd_config
```

## ARP

Переводит IP в MAC внутри локальной сети.

### arp -a
```
> показывает ARP таблицу — IP и MAC адреса устройств в сети
```

---

## ICMP

Протокол диагностики сети. Его использует ping.

### ping -c 4 ip
```
> отправить 4 пакета и остановиться
```

### ping -i 0.5 ip
```
> пинговать каждые 0.5 секунды
```

## Что происходит когда открываешь сайт

```
1. DNS: google.com → 142.250.74.46
2. TCP соединение на порт 443
3. TLS Handshake — шифрование
4. GET запрос по HTTP
5. Сервер отвечает 200 + HTML
6. Браузер отображает страницу
```
