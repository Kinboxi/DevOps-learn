## IP адрес

Адрес устройства в сети. Два типа:
- **Локальный** — внутри твоей сети `192.168.x.x`
- **Публичный** — уникальный в интернете `142.250.74.46`

> [!tip] Аналогия
> Локальный IP — номер комнаты в квартире. Публичный — адрес самого дома.

### ip a
```
> показывает все интерфейсы и IP адреса
```

### ifconfig
```
> альтернатива ip a
```

---

## MAC адрес

Физический адрес устройства, вшит в железо. Не меняется.

- MAC — работает внутри локальной сети
- IP — работает между сетями

> [!tip] Аналогия
> MAC — паспорт. IP — прописка.

---

## DHCP

Автоматически выдаёт IP адреса устройствам. Роутер = DHCP сервер в домашней сети.

> [!tip] Аналогия
> Роутер = администратор в отеле. Ты подключился — он выдал тебе номер комнаты.

---

## Маска подсети

Показывает где заканчивается сеть и начинается устройство.

```
IP:    192.168.0.101
Маска: 255.255.255.0

192.168.0 → сеть
101       → устройство
```

## DNS

Переводит доменное имя в IP адрес. `google.com` → `142.250.74.46`

```
8.8.8.8  — Google DNS
1.1.1.1  — Cloudflare DNS
```

### nslookup google.com
```
> узнать IP адрес домена
```

### dig google.com +short
```
> то же самое но короче
```

### cat /etc/resolv.conf
```
> какие DNS серверы использует система
```

## Порты

Порт — адрес конкретной программы на устройстве.

> [!tip] Аналогия
> IP — адрес здания. Порт — номер квартиры.

| Порт  | Протокол   |
|-------|------------|
| 22    | SSH        |
| 80    | HTTP       |
| 443   | HTTPS      |
| 53    | DNS        |
| 3306  | MySQL      |
| 5432  | PostgreSQL |
| 6379  | Redis      |
| 27017 | MongoDB    |

## Модель OSI

| Уровень | Название      | Пример         |
|---------|---------------|----------------|
| 7       | Приложение    | HTTP, SSH, DNS |
| 6       | Представление | TLS            |
| 5       | Сеанс         |                |
| 4       | Транспорт     | TCP, UDP       |
| 3       | Сеть          | IP             |
| 2       | Канальный     | MAC, Ethernet  |
| 1       | Физический    | кабель, WiFi   |

> [!warning] Траблшутинг — всегда снизу вверх
> Нет соединения (1) → не пингуется (3) → порт закрыт (4) → приложение упало (7)
