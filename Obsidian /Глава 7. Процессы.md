Процессы в Linux — программа которая запущена и выполняется прямо сейчас
Каждый процесс имеет свой уникальный номер — PID (Process ID)

---

## ps aux
Показывает все запущенные процессы
USER   PID  %CPU  %MEM  COMMAND
root     1   0.0   0.1  /sbin/init
user  1234   2.3   1.5  nginx
> a — все пользователи, u — подробно, x — включая фоновые

---

## top
Живой мониторинг процессов в реальном времени — обновляется каждую секунду
> нажми q чтобы выйти

## htop
То же самое что top но красивее и удобнее
> если нет — установи: apt install htop

---

## kill [PID]
Останавливает процесс по его номеру
> пример: kill 1234

## kill -9 [PID]
Принудительно убивает процесс — если обычный kill не помог
> пример: kill -9 1234

## pkill [имя]
Останавливает процесс по имени — не нужно искать PID

> пример: pkill nginx

---

## [команда] &
Запускает процесс в фоне — терминал остаётся свободным
> пример: python app.py &

## jobs
Показывает все фоновые процессы в текущей сессии

## fg
Возвращает фоновый процесс на передний план

---

## systemctl status [сервис]
Проверить состояние сервиса — запущен или нет
> пример: systemctl status nginx

## systemctl start [сервис]
Запустить сервис
> пример: systemctl start nginx

## systemctl stop [сервис]
Остановить сервис

## systemctl restart [сервис]
Перезапустить сервис — например после изменения конфига

## systemctl enable [сервис]
Сервис будет запускаться автоматически при старте системы

---

Зачем это DevOps
Сервер тормозит        → top / htop смотришь что жрёт ресурсы
Процесс завис          → kill -9
nginx не работает      → systemctl status nginx
После перезагрузки     → systemctl enable чтобы всё поднялось само
