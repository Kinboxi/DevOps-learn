# Git — это система контроля версий.
## Она помогает отслеживать изменения в проекте, возвращаться к предыдущим версиям и работать в команде без конфликтов.
---- 
# Что такое коммит(commit)? 
## Коммит — это снимок текущего состояния проекта

## Каждый коммит содержит:
- какие файлы были изменены,
- кем и когда,
- комментарий (сообщение коммита),
- уникальный идентификатор.
-----
# `git init` - Создает гит репозиторий (файл .git) в текущей папке, там храниться вся история проекта. 
---- 
# `git status
- **Показывает текущее состояние репозитория ,
- какие файлы были изменены,
- что добавлено в коммитт,
- что ещё нет.
---- 
# git add [имя_файла] — Добавляет изменения в индекс,  которые попадут в следующий коммит, файлы попадают в "staging area" 
## Можно добавить сразу все файлы
# `git add .`

--- 
# git commit -m "сообщение" 
## Создает коммит со всеми добавлеными файлами из индекса флаг -m дозволяет сразу написать сообщение.
