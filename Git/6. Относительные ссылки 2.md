**`~`** — переход на N шагов назад, удобнее чем писать много `^^^`

---

**Разница между `^` и `~`**
```
HEAD^   = один шаг назад (первый родитель)
HEAD^2  = второй родитель (только у merge-коммитов)
HEAD~3  = три шага назад по первой ветке
```
> Не знаешь есть ли merge — используй `~`, оно безопаснее

---

## `git checkout HEAD~N`
Переход на N коммитов назад
> пример: `git checkout HEAD~3`

---

## `git branch -f [ветка] HEAD~N`
Перемещает указатель ветки на старый коммит
> пример: `git branch -f main HEAD~3`

---

**Визуализация `git branch -f`**
```
до:    A──B──C──D──E  ← main (HEAD)

после git branch -f main HEAD~2:

       A──B──C  ← main
               \
                D──E  ← висят, ни одна ветка не ссылается
```

D и E не удалились физически — но через ветки их больше не видно

---

**Зачем нужно**
Быстро перемещаться по истории, откатывать ветки, тестировать старый код — без копипаста хешей
