**Отмена изменений в Git** — три разных инструмента для разных ситуаций

---

## `git reset HEAD~1`
Удаляет коммит, но изменения остаются в индексе (после `git add`) — то есть готовы к коммиту
> ⚠️ Меняет историю — не использовать в чужих ветках

## `git reset --hard HEAD~1`
Удаляет коммит и очищает индекс и рабочую папку — всё исчезает
> ⚠️ Безвозвратно

---

## `git revert HEAD`
Не трогает историю — создаёт новый коммит который отменяет изменения предыдущего
> Git откроет редактор → нажми `:wq` чтобы сохранить

---

## `git restore [файл]`
Файл ещё не в индексе (не делал `git add`) — возвращает его к состоянию последнего коммита
> пример: `git restore main.py`

## `git restore .`
То же самое но для всех файлов сразу

---

**Когда что использовать**
```
Закоммитил лишнее, хочу переделать     → git reset HEAD~1
Закоммитил лишнее, хочу удалить всё    → git reset --hard
Работаю в команде, нужен откат         → git revert
Изменил файл, ещё не делал add         → git restore
```
