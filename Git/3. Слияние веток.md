
---

**Слияние (merge)** — объединение изменений из одной ветки в другую
Пример: работал в `feature-login`, хочешь добавить в `main` — вместо копирования кода просто мержишь

---

### 📌 `git merge [имя_ветки]`
Вливает указанную ветку в текущую

> Сначала переключись в ту ветку **в которую** хочешь влить:
> ```
> git checkout main
> git merge bug-fix
> ```

---

## Merge-конфликт
Когда изменил одну строку в обеих ветках — Git не знает какой вариант оставить
```
<<<<<<< HEAD
текст из main
=======
текст из new-idea
>>>>>>> new-idea
```
Решение: открой → выбери нужный вариант → удали `<<<<<<<` `=======` `>>>>>>>` → сохрани → закоммить
```
git add имя_файла
git commit
```

---

## Fast-forward
Ветка просто "ушла вперёд" — Git перемещает указатель без нового коммита
```
A──B──C   →   A──B──C──D──E
```

## Merge commit
В обеих ветках есть изменения — создаётся новый объединяющий коммит

---

**Важно**
Merge не удаляет ветки — удали вручную после слияния:

### 📌 `git branch -d feature-login`
Порядок: финальный коммит в своей ветке → потом merge
