# Слияние(merge) — это объединение изменений из одной ветки в другую
Пример: ты работал над функцией в ветке feature-login, а теперь хочешь добавить эту функцию в основной проект (main).
# Вместо того чтобы копировать код вручную, ты просто объединяешь ветки через `git merge`
---- 
# `git merge [имя_ветки]` —Объединяет указанную ветку в текущую ветку.



---- 


## Важно: нужно находиться в той ветке, в которую хочешь влить изменения.
Пример : 
# `git checkout main`
# `git merge bug-fix`


-----

# Merge-конфликт — когда Git не может сам объединить изменения
# Пример: ты изменил одну и ту же строку и в `main`, и в `new-idea` — Git не знает, какой вариант оставить

# Как выглядит конфликт в файле
```
<<<<<<< HEAD
текст из текущей ветки (main)
=======
текст из вливаемой ветки (new-idea)
>>>>>>> new-idea
```

# Как решить: открой файл → выбери нужный вариант → удали спецсимволы `<<<<<<<` `=======` `>>>>>>>` → сохрани → закоммить
```
git add имя_файла
git commit
```
----
# Два типа merge

# Fast-forward — если ветка просто "ушла вперёд", Git перемещает указатель без нового коммита
```
A──B──C   →   A──B──C──D──E
```
# Merge commit — если в обеих ветках есть изменения, создаётся новый объединяющий коммит

----
# Merge не удаляет ветки — после слияния удали вручную если не нужна
```
git branch -d feature-login
```
# Правильный порядок: сначала финальный коммит в своей ветке → потом merge
