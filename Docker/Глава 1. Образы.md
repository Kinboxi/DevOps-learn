# Глава 1. Docker — Образы (Images)

> [!info] Зачем это знать
> Образ — основа всего Docker. Без понимания образов не поймёшь контейнеры и всё остальное.

---

## Что такое образ

Образ — это **неизменяемый файл-шаблон**, в котором упаковано всё что нужно программе:
- Урезанная операционная система
- Язык программирования
- Библиотеки и зависимости
- Код приложения

> [!tip] Аналогия
> Образ — это установочный диск. Сам по себе не работает — но из него можно создать контейнер.

---

## Важные правила

> [!warning] Образ нельзя изменить
> Образ **неизменяемый**. Хочешь изменить — пересобираешь заново.

> [!info] Из одного образа — много контейнеров
> Один образ может быть источником для сотен контейнеров одновременно.

---

## Слои образа

Образ состоит из **слоёв**. Каждый слой — это часть содержимого образа.

```
Слой 3 — Flask
Слой 2 — Python 3.10
Слой 1 — Ubuntu (база)
```

**Зачем слои:**
- Общие слои используются между образами — экономия места на диске
- При пересборке — пересобираются только изменённые слои — экономия времени

---

## Где хранятся образы

```
/var/lib/docker/
```

Docker управляет этой папкой сам.

---

## Команды

### Посмотреть все образы
```bash
docker images
```

Выводит список всех образов на твоём компьютере:
- название
- тег (версия)
- ID образа
- дата создания
- размер

### Подробная информация об образе
```bash
docker image inspect <image_name>
```

Выводит метаданные образа:
- слои
- переменные окружения
- открытые порты
- команда запуска
