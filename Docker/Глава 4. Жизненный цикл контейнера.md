# Глава 4. Docker — Жизненный цикл контейнера (Container Lifecycle)

> [!info] Зачем это знать
> Это основа работы с контейнерами каждый день. Без этого не сможешь запускать, останавливать и управлять контейнерами.

---

## Жизненный цикл контейнера

Контейнер проходит через несколько состояний:

```
created → running → paused → stopped → removed
```

| Состояние | Что происходит |
|---|---|
| created | Контейнер создан но не запущен |
| running | Контейнер работает |
| paused | Контейнер заморожен, процессы на паузе |
| stopped | Контейнер остановлен, данные ещё есть |
| removed | Контейнер удалён полностью |

---

## docker create

Создаёт контейнер из образа — но **не запускает** его.

```bash
docker create nginx
```

Контейнер готов но спит. Можно настроить и запустить позже.

> [!tip] Когда использовать
> Когда хочешь подготовить контейнер заранее и запустить в нужный момент.

---

## docker start

Запускает уже созданный контейнер.

```bash
docker start <container_id>
```

```bash
# Запустить и видеть вывод в терминале
docker start -a <container_id>
# -a (attach) — подключиться к выводу контейнера
```

> [!tip] Разница с docker run
> - `docker run` = create + start в одной команде
> - `docker start` — только запускает уже существующий контейнер

---

## docker run

Самая частая команда. Создаёт и сразу запускает контейнер.

```bash
docker run nginx
```

```bash
# Запустить в фоне
docker run -d nginx

# Запустить с именем
docker run -d --name myserver nginx

# Запустить с пробросом порта
docker run -d -p 8080:80 nginx
# 8080 — порт на твоём компьютере
# 80 — порт внутри контейнера

# Запустить и зайти внутрь
docker run -it ubuntu bash
# -i — интерактивный режим
# -t — терминал внутри

# Запустить и удалить после остановки
docker run --rm nginx
# --rm — контейнер удалится сам когда остановится
```

> [!tip] Аналогия
> `docker run` — это как нажать кнопку "Пуск" на машине. Она и заводится и едет.

---

## docker stop

Останавливает контейнер **мягко**. Даёт процессам время завершиться (10 секунд по умолчанию).

```bash
docker stop <container_id>
```

```bash
# Остановить с таймаутом 5 секунд
docker stop -t 5 <container_id>
```

> [!info] Что происходит внутри
> Docker отправляет сигнал **SIGTERM** — "пожалуйста завершись". Если за 10 секунд не завершился — отправляет **SIGKILL** — "умри немедленно".

---

## docker kill

Останавливает контейнер **жёстко**. Немедленно без ожидания.

```bash
docker kill <container_id>
```

> [!warning] Осторожно
> `docker kill` не даёт процессам завершиться корректно. Используй только если `docker stop` не помогает или контейнер завис.

| | docker stop | docker kill |
|---|---|---|
| Скорость | Медленнее | Мгновенно |
| Безопасность | Мягко, корректно | Жёстко |
| Сигнал | SIGTERM → SIGKILL | SIGKILL |

---

## docker rm

Удаляет остановленный контейнер.

```bash
docker rm <container_id>
```

```bash
# Удалить принудительно (даже запущенный)
docker rm -f <container_id>

# Удалить все остановленные контейнеры
docker container prune
```

> [!warning] Важно
> `docker rm` удаляет контейнер — но не образ. Образ остаётся на диске.

---

## docker pause / unpause

Замораживает контейнер. Процессы внутри останавливаются но не завершаются.

```bash
# Заморозить
docker pause <container_id>

# Разморозить
docker unpause <container_id>
```

> [!tip] Аналогия
> Как поставить игру на паузу. Всё замерло — но не закрылось. Снял паузу — продолжаешь с того же места.

> [!tip] Когда использовать
> Когда нужно временно освободить ресурсы CPU но не останавливать контейнер полностью.

---

## docker restart

Останавливает и снова запускает контейнер.

```bash
docker restart <container_id>
```

```bash
# Перезапустить с таймаутом
docker restart -t 5 <container_id>
```

> [!tip] Когда использовать
> Когда контейнер завис или приложение внутри перестало отвечать.

---

## Полная шпаргалка команд

```bash
docker create nginx                    # создать контейнер
docker start <id>                      # запустить контейнер
docker run -d --name myapp nginx       # создать и запустить
docker stop <id>                       # остановить мягко
docker kill <id>                       # остановить жёстко
docker pause <id>                      # заморозить
docker unpause <id>                    # разморозить
docker restart <id>                    # перезапустить
docker rm <id>                         # удалить
docker rm -f <id>                      # удалить принудительно
docker container prune                 # удалить все остановленные
docker ps                              # список запущенных
docker ps -a                           # список всех контейнеров
```

---

> [!summary] Итог
> Контейнер живёт по циклу: **create → start → pause/stop → restart → rm**. Для мягкой остановки используй `stop`. Для жёсткой — `kill`. Данные внутри пропадают при удалении — для хранения нужны Volumes.
>
