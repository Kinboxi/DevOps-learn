# Глава 3. Docker — Контейнеры (Containers)

> [!info] Зачем это знать
> Понять как работают контейнеры — основа всего Docker.

---

## Что такое контейнер

Контейнер — это **изолированный процесс** на твоей операционной системе.

Запустил контейнер — на твоей системе появился новый процесс. Остановил — процесс исчез.

> [!tip] Аналогия
> Контейнер — это комната в общей квартире. У каждого своя комната — но фундамент и электричество общие.

---

## Как контейнер изолирован

Docker использует механизм Linux — **Namespaces**.

Контейнер не видит:
- Процессы других контейнеров
- Сеть других контейнеров
- Файлы других контейнеров

Docker настраивает namespaces автоматически — тебе не нужно делать это вручную.

---

## Контейнер vs Виртуальная машина

| | Контейнер | Виртуальная машина |
|---|---|---|
| Своя OS | ❌ Использует ядро хоста | ✅ Своя полная OS |
| Вес | Мегабайты | Гигабайты |
| Скорость запуска | Секунды | Минуты |

---

## Как контейнер связан с образом

```
Image (неизменяемый)
    ↓           ↓
Контейнер 1   Контейнер 2
(свои         (свои
изменения)    изменения)
```

- Из одного образа можно создать много контейнеров
- Каждый контейнер живёт своей жизнью — не видит других
- Image никогда не меняется

---

## Важно про данные

> [!warning] Данные пропадают
> Всё что ты делаешь внутри контейнера — живёт только в нём.
> Удалил контейнер — данные пропали. Image остался нетронутым.
